Relazione sullo sviluppo collaborativo con GitHub

Cos’è GitHub

GitHub è una piattaforma online (https://github.com) che serve principalmente per ospitare, condividere e collaborare su progetti di software. È uno strumento fondamentale nel mondo della programmazione moderna perché permette a più persone di lavorare contemporaneamente sullo stesso progetto, anche da luoghi diversi.

Cos’è in pratica

GitHub è basato su Git, un sistema di controllo di versione creato da Linus Torvalds (lo stesso che ha sviluppato Linux). Git tiene traccia di tutte le modifiche fatte ai file di un progetto, permettendo di:

  • Tornare a versioni precedenti del codice
  • Confrontare modifiche
  • Lavorare in gruppo senza rischiare di sovrascrivere il lavoro altrui

In sostanza, GitHub unisce le funzioni di Git a un’interfaccia web semplice e collaborativa, rendendo il lavoro di gruppo più organizzato e trasparente.

Cosa puoi fare su GitHub

  • Salvare codice online in un repository (una cartella del progetto)
  • Collaborare con altre persone, integrando e revisionando il loro codice (pull requests, issues)
  • Gestire versioni del progetto con Git
  • Creare siti web statici con GitHub Pages
  • Usare GitHub Actions per automatizzare processi come test, aggiornamenti e pubblicazioni

Come funziona GitHub

1. Git + GitHub: la coppia base

  • Git è il programma che tiene traccia delle modifiche ai file sul tuo computer.
  • GitHub è il sito dove puoi caricare online i tuoi progetti Git per condividerli o collaborare con altri.

2. Come funziona in pratica (passaggi base)

Step 1: Crei un account su GitHub

Vai su github.com e ti registri gratuitamente. Puoi creare progetti pubblici (visibili a tutti) o privati (solo per il tuo gruppo).

Step 2: Crei un repository

Un repository (abbreviato “repo”) è una cartella che contiene tutto il progetto: codice, documenti, immagini, ecc. Puoi:

  • Crearlo direttamente sul sito GitHub, oppure
  • Crearlo in locale con Git e poi caricarlo online.

Step 3: Fai modifiche e le “committi”

Ogni volta che modifichi un file:

  1. git add → prepari i file da salvare
  2. git commit → registri la modifica con un messaggio
  3. git push → invii tutto su GitHub

Step 4: Collabori

Altri membri del team possono:

  • Clonare il progetto
  • Fare modifiche
  • Inviare una pull request (una proposta di aggiornamento). Il responsabile del progetto può poi esaminarla e approvarla.

Step 5: Gestisci problemi o miglioramenti

Si possono aprire issue per segnalare problemi o idee, organizzare milestone per obiettivi e creare rami (branches) per sviluppare nuove funzioni senza modificare il codice principale.

Cos’è HTML

HTML significa HyperText Markup Language, cioè Linguaggio di marcatura per ipertesti. È il linguaggio base del web, utilizzato per costruire la struttura di qualsiasi pagina internet. Serve a indicare al browser come deve essere organizzato e visualizzato il contenuto (testo, immagini, link, video, ecc.).

Come funziona HTML

HTML non è un linguaggio di programmazione come Python o JavaScript, ma un linguaggio di struttura. Serve per organizzare i contenuti di una pagina attraverso l’uso dei tag, cioè parole chiave racchiuse tra parentesi angolari < >.

Ogni elemento HTML è composto da:

  • Un tag di apertura → <tag>
  • Un tag di chiusura → </tag>
  • Tra questi due tag c’è il contenuto.

Ecco alcuni dei principali tag HTML:

Tag Funzione
<html>Apre la pagina HTML
<head>Contiene informazioni invisibili come titolo e stili
<body>Contiene il contenuto visibile
<h1>Titolo principale
<p>Paragrafo di testo
<img>Inserisce un’immagine

Come si visualizza una pagina HTML

Per creare e visualizzare una pagina HTML servono solo pochi passaggi:

  1. Apri un editor di testo, come Blocco Note, Visual Studio Code o Notepad++.
  2. Scrivi il codice HTML, ad esempio:
<!DOCTYPE html>
<html>
  <head>
    <title>La mia prima pagina</title>
  </head>
  <body>
    <h1>Benvenuti alla Pizzeria del Team!</h1>
    <p>Questo è il nostro menù digitale realizzato con HTML e GitHub.</p>
    <img src="pizza.jpg" alt="Una pizza gustosa">
  </body>
</html>

Salva il file con estensione .html (es. menu.html).

Aprilo con un browser (Chrome, Edge, Firefox, Safari...).

Il browser leggerà il codice e mostrerà la pagina formattata e leggibile, senza mostrare i tag. In questo modo, da un semplice file di testo nasce una pagina web vera e propria.

Puoi poi arricchirla con CSS (per i colori, font e stili grafici) e JavaScript (per l’interattività).
